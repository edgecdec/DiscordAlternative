- The app's PrismaClient in `src/lib/db.ts` used `path.join(process.cwd(), 'data', 'discord.db')` which correctly resolved to the project's `data/` dir. Mismatch meant migrations went to wrong file.
- Fix was already applied by human in commit 5fedd2c: changed to `file:./data/discord.db`.
- Verified: deploy webhook ran, migrations applied to correct DB, register endpoint returns valid user.
- Confirmed deploy_webhook.sh will work correctly on future deploys.
- Lessons: In Prisma 7, datasource URL in `prisma.config.ts` resolves relative to the config file location, NOT the schema file. Always verify the runtime DB path matches the migration DB path.
